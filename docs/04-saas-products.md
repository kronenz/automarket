# AutoMarket 12개 SaaS 제품 카탈로그

> **TL;DR**
> 1. AutoMarket의 4개 Core Node는 각각 독립 판매 가능한 SaaS 제품으로 분해된다 — 기회 발견 3개, 고객 인텔리전스 3개, 콘텐츠 공급망 4개, 채널 오케스트레이터 2개.
> 2. 각 제품은 "단독 가치 테스트"를 통과해야 하며, 연결 시 단독 대비 2배 이상의 시너지를 제공하도록 설계한다.
> 3. 독립성 평가 결과: **PageFactory·TrendRadar·ReviewOps·ChurnGuard**가 강한 독립 가치를 보유하고, **MarginSim·BrandForge**는 독립 제품보다 다른 제품의 기능으로 흡수하는 방향이 유리하다.

---

## SaaS 독립화 기준 (4가지)

각 제품은 아래 4가지 기준으로 독립 SaaS 적합성을 평가한다.

| 기준 | 질문 |
|------|------|
| **단독 가치** | "이것만 써도 월 N만원 낼 만한가?" |
| **타겟 명확성** | "누가 쓰는가?" 한 문장으로 답할 수 있는가 |
| **데이터 자급자족** | 다른 노드 없이 자체 데이터로 동작하는가 |
| **통합 시 증폭** | 연결하면 단독 대비 2배 이상 가치가 발생하는가 |

---

## Node 1 — 기회 발견 엔진 (Opportunity Discovery)

Node 1의 역할: 시장 신호 수집 → 상품 후보 추출 → 수익성 검증을 단일 루프로 처리. 상세 아키텍처는 [./02-architecture.md](./02-architecture.md) 참고.

---

### 1. TrendRadar — "팔릴 것을 먼저 안다"

| 항목 | 내용 |
|------|------|
| **타겟** | 온라인 셀러, MD, 상품기획자 |
| **핵심 기능** | 트렌드 크롤링, 키워드 수요 예측, 기회 스코어링, 경쟁 강도 측정 |
| **단독 가치** | 강 — 트렌드 리포트만으로 월 9.9만원 즉시 지불 의향 존재 |
| **해자 (Moat)** | 네이버·쿠팡·SNS 공개 데이터 수집 파이프라인 누적 |
| **경쟁 참고** | 셀러마스터, 아이템스카우트 (기능 유사, 이커머스 전략 연동 없음) |

**MVP 스코프 (3개)**
1. 카테고리별 트렌드 크롤링 + 일간 리포트
2. 키워드 수요 예측 (검색량 추이 + 계절성)
3. 기회 스코어 자동 계산 (수요 × 경쟁 역수)

**기술 스택** Python/FastAPI + 크롤러 + ML 예측 모델 + PostgreSQL

**수익 모델** 구독 3티어 — Starter 9.9만원/월, Pro 29.9만원/월, Enterprise 99만원/월

**통합 시 시너지** TrendRadar 기회 스코어가 Arbitration Layer를 통해 PageFactory로 전달되면, 수요가 검증된 상품의 상세페이지를 우선 생성하는 자동 루프가 완성된다.

---

### 2. SourceMatch — "최적 공급처를 자동으로 연결한다"

| 항목 | 내용 |
|------|------|
| **타겟** | 소싱이 필요한 셀러, 초기 브랜드 창업자 |
| **핵심 기능** | 공급처 DB 검색, 조건 비교(MOQ·리드타임·단가), 견적 자동화 |
| **단독 가치** | 중 — 소싱 경험이 없는 셀러에게 명확한 가치, 숙련 셀러는 이미 네트워크 보유 |
| **해자** | 공급처 DB 네트워크 효과 (등록 공급처 수 = 진입 장벽) |
| **경쟁 참고** | 알리바바(글로벌), 오픈마켓 소싱 카페(비정형) |

**MVP 스코프 (3개)**
1. 공급처 DB 구축 + 카테고리별 검색
2. MOQ·리드타임·단가 조건 비교 테이블
3. 견적 요청 자동화 (이메일/메시지 템플릿 발송)

**기술 스택** Node.js + PostgreSQL + 이메일 자동화

**수익 모델** 프리미엄 — 월 5건 무료 매칭, 초과 건당 과금 또는 월 구독

**통합 시 시너지** TrendRadar 기회 스코어 상위 상품에 대해 SourceMatch가 공급처를 자동 추천하면, 기회 발견 → 소싱까지의 TTR(Time-to-Revenue)이 대폭 단축된다.

---

### 3. MarginSim — "채널별 마진을 한 화면에서 시뮬레이션한다"

| 항목 | 내용 |
|------|------|
| **타겟** | 멀티채널 셀러, 브랜드 재무 담당자 |
| **핵심 기능** | 원가·물류·수수료 통합 마진 계산, 채널별 비교, Go/No-go 판단 |
| **단독 가치** | 약 — 엑셀로 대체 가능, 1회성 사용 패턴 |
| **해자** | 채널 수수료 DB 실시간 업데이트 (쿠팡·네이버·11번가 등) |
| **경쟁 참고** | 자체 엑셀, 셀러툴 내장 계산기 |

> **독립 제품 리스크**: 단독 반복 사용 동인이 약하다. TrendRadar 또는 ChannelSync의 내장 기능으로 흡수하는 방향이 현실적이다.

**MVP 스코프 (3개)**
1. 채널별 수수료 DB + 자동 업데이트
2. 원가·물류·마진 통합 시뮬레이션
3. 채널별 마진 비교 + Go/No-go 기준선

**기술 스택** React + 계산 엔진 + 채널 수수료 DB

**수익 모델** 프리미엄 (TrendRadar 번들 또는 독립 구독)

**통합 시 시너지** TrendRadar 기회 스코어 상위 상품의 마진을 자동 계산해 결과를 Arbitration Layer에 전달하면, 수익성 검증까지 완전 자동화된 Node 1 루프가 완성된다.

## Node 2 — 고객 인텔리전스 (Customer Intelligence)

Node 2는 파이프라인 직렬 구조에 포함되지 않는 **cross-cutting 서비스 레이어**다. 나머지 3개 노드 모두에 페르소나·LTV·이탈 신호를 실시간 주입한다. 상세 설계는 [./02-architecture.md](./02-architecture.md) 참고.

---

### 4. PersonaAI — "데이터가 고객을 그린다"

| 항목 | 내용 |
|------|------|
| **타겟** | 마케터, 브랜드매니저, D2C 운영팀 |
| **핵심 기능** | 자동 페르소나 생성, 세그먼트 빌더, 코호트 분석 |
| **단독 가치** | 강 — 기존 고객 데이터만으로 즉시 인사이트 제공 |
| **해자** | 스마트스토어·쿠팡 주문 데이터 직접 연동 (채널톡은 CS 중심, 판매 전략 연동 없음) |
| **경쟁 참고** | 채널톡(CS 중심), Amplitude(웹 행동 중심) |

**MVP 스코프 (3개)**
1. 주문 데이터 연동 (스마트스토어 / 쿠팡 API)
2. 자동 세그먼트 분류 (ML 클러스터링)
3. 페르소나 카드 자동 생성 + 내보내기

**기술 스택** Python + ML 클러스터링 (k-means / DBSCAN) + React 대시보드

**수익 모델** 구독 — 연동 채널 수 + 세그먼트 수 기준 티어

**통합 시 시너지** PersonaAI 세그먼트가 PageFactory에 전달되면 세그먼트별 맞춤 상세페이지를, AdPilot에 전달되면 세그먼트별 광고 예산 배분을 자동화한다.

---

### 5. BrandForge — "AI가 브랜드 정체성을 설계한다"

| 항목 | 내용 |
|------|------|
| **타겟** | 신규 브랜드 창업자, D2C 런칭팀 |
| **핵심 기능** | 톤앤매너·컬러·메시징 생성, 포지셔닝 맵, 경쟁사 분석 |
| **단독 가치** | 약 — 1회성 니즈 강함, 반복 사용 동인 낮음, TAM 협소 |
| **해자** | 경쟁사 포지셔닝 데이터 누적 |
| **경쟁 참고** | 로고 AI 툴 (범용), 브랜딩 에이전시 (고비용) |

> **독립 제품 리스크**: 브랜드 가이드라인 생성은 1회성 작업으로 구독 반복 결제 명분이 약하다. PageFactory 내 "브랜드 설정" 모듈로 통합하거나, 건당 과금 서비스로 포지셔닝하는 것이 현실적이다.

**MVP 스코프 (2개)**
1. 브랜드 가이드 자동 생성 (LLM 기반 톤앤매너 + 컬러 팔레트)
2. 포지셔닝 맵 + 경쟁사 비교

**기술 스택** LLM API (GPT-4o / Claude) + React + 디자인 토큰 엔진

**수익 모델** 건당 과금 (가이드 1세트 N만원) 또는 연간 구독

**통합 시 시너지** BrandForge 가이드라인이 PageFactory 템플릿과 SocialSpray 콘텐츠에 적용되면, 모든 접점에서 브랜드 일관성이 자동으로 유지된다.

---

### 6. ChurnGuard — "떠나기 전에 잡는다"

| 항목 | 내용 |
|------|------|
| **타겟** | 자사몰·D2C 운영사, 구독 서비스 |
| **핵심 기능** | LTV 예측, 이탈 경보, 재구매 자동화 |
| **단독 가치** | 강 — 이탈 방지 1건이 CAC 전체를 회수, ROI 직결 |
| **해자** | 주문·방문 데이터 기반 예측 모델 고도화 (데이터 누적 = 정확도 향상) |
| **경쟁 참고** | CRM 툴(Braze, 채널톡) — 이탈 예측 없이 발송만 지원 |

**MVP 스코프 (3개)**
1. 이탈 예측 모델 (최근 구매일·빈도·금액 기반)
2. 이탈 위험 고객 알림 대시보드
3. 재구매 쿠폰 자동 발송 (이메일/카카오톡)

**기술 스택** Python + ML 예측 모델 (Random Forest / XGBoost) + 메시지 자동화 API

**수익 모델** 구독 — 활성 고객 수 기준 티어

**통합 시 시너지** ChurnGuard 이탈 신호가 Arbitration Layer를 통해 Node 3에 전달되면, 이탈 위험 고객에게 맞춤 콘텐츠와 쿠폰을 자동 생성·발송하는 완전 자동화 루프가 형성된다.

## Node 3 — 콘텐츠 공급망 (Content Supply Chain)

Node 3의 역할: AI Generator(내부 생산)와 Partner Network(외부 파트너)의 듀얼 엔진으로 콘텐츠 패키지를 생성. 상세 설계는 [./02-architecture.md](./02-architecture.md) 참고.

---

### 7. PageFactory — "상품 정보를 넣으면 상세페이지가 나온다"

> **최강 PMF (Product-Market Fit)** — Phase 1 MVP 최우선 출시 대상

| 항목 | 내용 |
|------|------|
| **타겟** | 모든 온라인 셀러 (이커머스 사업자라면 누구나) |
| **핵심 기능** | 상품 정보 → 상세페이지 자동 생성, 템플릿 선택, A/B 테스트, 채널별 최적화 |
| **단독 가치** | 강 — 상세페이지 제작 비용(외주 30~100만원)과 비교 시 즉각 납득 |
| **해자** | 이커머스 전문 AI 엔진 (범용 디자인 툴과 차별화) |
| **경쟁 참고** | 미리캔버스·캔바(범용 디자인), Jasper(카피 전용) |

**MVP 스코프 (4개)**
1. 상품명·특징·이미지 입력 → 상세페이지 HTML/이미지 자동 생성
2. 이커머스 특화 템플릿 3종 (쿠팡형 / 스마트스토어형 / 브랜드 D2C형)
3. 채널별 규격 자동 최적화 (이미지 크기·글자수 제한)
4. A/B 변형 2종 자동 생성

**기술 스택** LLM API + 이미지 생성 모델 (DALL-E / Stable Diffusion) + 템플릿 엔진 + Next.js

**수익 모델** 프리미엄 — 월 5건 무료, 초과 건당 과금 또는 월 구독 (Unlimited)

**통합 시 시너지** TrendRadar 기회 스코어 상위 상품을 자동 수신해 상세페이지를 우선 생성하고, PersonaAI 세그먼트별로 버전을 분기하면 완전 자동화 콘텐츠 파이프라인이 완성된다.

---

### 8. SocialSpray — "하나의 소스로 모든 채널에 뿌린다"

| 항목 | 내용 |
|------|------|
| **타겟** | SNS 마케팅 담당 셀러, 소규모 마케팅팀 |
| **핵심 기능** | 원소스 멀티포맷 변환, 예약 배포, 성과 트래킹 |
| **단독 가치** | 중 — 명확하나 글로벌 경쟁 제품 대비 차별화 포인트 필요 |
| **해자** | 채널 API 생태계 연동 깊이 (한국 채널 특화: 네이버 블로그, 카카오) |
| **경쟁 참고** | Buffer, Hootsuite (글로벌, 한국 채널 미지원), 에어노티 |

> **글로벌 경쟁 과밀 주의**: Buffer·Hootsuite 대비 한국 이커머스 채널(네이버 블로그·카카오채널·스마트스토어 소식) 특화로 차별화해야 생존 가능하다.

**MVP 스코프 (3개)**
1. 인스타그램·네이버 블로그 포맷 자동 변환
2. 예약 배포 스케줄러
3. 채널별 성과(도달·클릭·전환) 통합 대시보드

**기술 스택** Node.js + 채널 API (Instagram Graph, Naver Blog, Kakao) + Bull Queue(예약)

**수익 모델** 구독 — 연동 채널 수 + 월 발행 건수 기준 티어

**통합 시 시너지** PageFactory에서 생성된 상세페이지 콘텐츠를 SocialSpray가 SNS 포맷으로 자동 변환·배포하면, 콘텐츠 생산부터 배포까지 사람 개입 없이 연결된다.

---

### 9. ReviewOps — "체험단과 리뷰를 시스템으로 운영한다"

> **시장 공백 확인** — 체험단 운영 전문 SaaS 부재, Phase 1 MVP 대상

| 항목 | 내용 |
|------|------|
| **타겟** | 체험단 운영 셀러, 마케팅 대행사, 상품 리뷰 관리 담당자 |
| **핵심 기능** | 리뷰어 DB, 체험단 모집·관리, 협력사 발주 관리, 품질 스코어링 |
| **단독 가치** | 강 — 체험단 운영 자동화만으로 대행사 인건비 대비 즉각 ROI |
| **해자** | 리뷰어 DB 네트워크 효과 (리뷰어 수 = 플랫폼 가치) |
| **경�급 참고** | 레뷰(한국, 인플루언서 중심), 리뷰어 카페(비정형) |

**MVP 스코프 (3개)**
1. 리뷰어 DB 구축 + 카테고리·채널별 검색
2. 체험단 모집 → 발송 → 완료 확인 워크플로
3. 리뷰 품질 스코어링 (키워드 포함률·이미지 품질·길이)

**기술 스택** Node.js + PostgreSQL + 알림 자동화 (카카오·이메일)

**수익 모델** 트랜잭션 — 체험단 1건당 수수료 또는 월 구독 (건수 제한)

**통합 시 시너지** ReviewOps 리뷰 감성 데이터가 Node 2(PersonaAI, ChurnGuard)에 입력되면, 리뷰 악화 시 자동으로 이탈 방지 캠페인이 트리거된다.

---

### 10. InfluBridge — "ROI가 검증된 인플루언서만 연결한다"

| 항목 | 내용 |
|------|------|
| **타겟** | 인플루언서 마케팅을 운영하는 브랜드, 마케팅 대행사 |
| **핵심 기능** | AI 매칭, 캠페인 관리, ROI 트래킹 |
| **단독 가치** | 중-강 — 인플루언서 캠페인 ROI 측정 자체가 시장 니즈 |
| **해자** | SNS 크롤링 기반 인플루언서 성과 DB + 캠페인 결과 누적 |
| **경쟁 참고** | 레뷰·픽업(매칭 중심, ROI 트래킹 미흡) |

**MVP 스코프 (3개)**
1. 인플루언서 DB + AI 카테고리·팔로워·평균 참여율 매칭
2. 캠페인 생성·진행·완료 관리 워크플로
3. 채널별 ROI 트래킹 (노출·클릭·전환 연동)

**기술 스택** Python + SNS 크롤러 + React + UTM 추적

**수익 모델** 구독 + 성과 수수료 (캠페인 전환 매출의 N%)

**통합 시 시너지** InfluBridge ROI 데이터가 AdPilot에 전달되면, 성과 높은 인플루언서 채널에 광고 예산을 자동으로 추가 배분한다.

## Node 4 — 채널 오케스트레이터 (Channel Orchestrator)

Node 4의 역할: 유통 채널 통합 + 마케팅 자동화 실행. AI 자동화 목표 85%, TTR 1시간. 상세 설계는 [./02-architecture.md](./02-architecture.md) 참고.

### 11. ChannelSync — "재고와 주문을 채널 경계 없이 통합한다"

| 항목 | 내용 |
|------|------|
| **타겟** | 2개 이상 채널에서 판매하는 멀티채널 셀러 |
| **핵심 기능** | 쿠팡·스마트스토어·11번가 통합, 재고 동기화, 주문 통합 관리 |
| **단독 가치** | 강 — 멀티채널 셀러의 재고 사고·누락 주문 방지 = 직접 손실 차단 |
| **해자** | 채널 API 연동 깊이와 안정성 (인증·갱신·예외처리 노하우) |
| **경쟁 참고** | 사방넷, 플레이오토 (주문·재고 중심, 마케팅 연동 없음) |

**MVP 스코프 (3개)**
1. 쿠팡·스마트스토어 2채널 재고 실시간 동기화
2. 통합 주문 수신·처리·상태 관리
3. 채널별 매출 통합 대시보드

**기술 스택** Node.js + 채널 API (Coupang Open API, Naver Commerce API) + PostgreSQL + WebSocket

**수익 모델** 구독 — 연동 채널 수 + 월 주문 건수 기준 티어

**통합 시 시너지** ChannelSync 재고 신호가 AdPilot에 전달되면 재고 소진 임박 상품의 광고를 자동 중단하고, TrendRadar에 전달되면 판매 속도 기반 기회 재스코어링이 자동화된다.

---

### 12. AdPilot — "ROAS가 목표를 밑돌면 예산이 스스로 이동한다"

| 항목 | 내용 |
|------|------|
| **타겟** | 퍼포먼스 마케터, 광고 예산을 직접 운영하는 셀러·브랜드 |
| **핵심 기능** | ROAS 기반 예산 자동 배분, 재고 연동 광고 on/off, 크로스채널 대시보드 |
| **단독 가치** | 강 — 광고 낭비 차단 = 광고비 절감 = 직접 ROI |
| **해자** | 광고 API + 매출·재고 DB 연동 (광고만 최적화하는 툴과 차별화) |
| **경쟁 참고** | 네이버 성과형 광고 자동화(단일 채널), Adline(글로벌) |

**MVP 스코프 (4개)**
1. 네이버·쿠팡 광고 API 연동
2. ROAS 실시간 모니터링 대시보드
3. ROAS 목표 대비 자동 예산 배분
4. 재고 0 상품 광고 자동 중단

**기술 스택** Python + 광고 API (Naver SA, Coupang Ads) + React + 스케줄러

**수익 모델** 구독 + 광고비 연동 수수료 (관리 광고비의 N%)

**통합 시 시너지** AdPilot이 ChannelSync 재고·PersonaAI 세그먼트·TrendRadar 트렌드 스코어를 동시에 수신하면, 재고·고객·시장 3개 축을 통합한 광고 자동 최적화가 가능해진다.

12개 제품의 SaaS 독립화 4기준을 한눈에 비교한다. **강 / 중 / 약** 3단계로 표시.

| 제품 | 단독 가치 | 타겟 명확성 | 데이터 자급자족 | 통합 시 증폭 | **종합** |
|------|-----------|-------------|-----------------|-------------|----------|
| TrendRadar | 강 | 강 | 강 | 강 | **독립 출시 적합** |
| SourceMatch | 중 | 강 | 중 | 강 | **조건부 독립** |
| MarginSim | 약 | 중 | 강 | 중 | **통합 흡수 권장** |
| PersonaAI | 강 | 강 | 중 | 강 | **독립 출시 적합** |
| BrandForge | 약 | 중 | 중 | 중 | **통합 흡수 권장** |
| ChurnGuard | 강 | 강 | 중 | 강 | **독립 출시 적합** |
| PageFactory | 강 | 강 | 강 | 강 | **최우선 독립 출시** |
| SocialSpray | 중 | 강 | 중 | 강 | **조건부 독립** |
| ReviewOps | 강 | 강 | 강 | 강 | **독립 출시 적합** |
| InfluBridge | 중 | 강 | 중 | 강 | **조건부 독립** |
| ChannelSync | 강 | 강 | 강 | 강 | **독립 출시 적합** |
| AdPilot | 강 | 강 | 중 | 강 | **독립 출시 적합** |

**판정 기준**
- **최우선 독립 출시**: 4기준 모두 강 — Phase 1 MVP 대상
- **독립 출시 적합**: 3기준 이상 강 — Phase 2~3 대상
- **조건부 독립**: 단독 가치는 있으나 차별화 전략 필요
- **통합 흡수 권장**: 독립 반복 사용 동인 부족, 인접 제품 기능으로 통합
